## File Descriptions

### 1. SKILL.md
**Purpose:** Apply Clay security best practices for API key management, data access control, and prospect data protection. Covers secure configuration patterns and audit logging.
**Workflow:** Security hardening skill. Use during initial setup and security audits.
**Relates to:** Builds on clay-install-auth; prerequisite for clay-data-handling and clay-enterprise-rbac.

## Summary

This skill implements security best practices for Clay integrations. It covers secure API key storage (environment variables, secret managers), access control patterns for team environments, audit logging for compliance, secure handling of enriched prospect data (emails, phone numbers, company info), and preventing accidental key exposure in logs or error messages.
