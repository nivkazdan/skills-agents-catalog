## File Descriptions

### 1. SKILL.md
**Purpose:** Configure Clay enterprise team access, role-based permissions, and organization management. Implements enterprise-grade access control.
**Workflow:** Enterprise security skill. Use when setting up Clay for enterprise teams.
**Relates to:** Builds on clay-multi-env-setup; integrates with clay-data-handling for data access control.

## Summary

This skill implements enterprise RBAC for Clay integrations. It covers defining roles (admin, editor, viewer) with appropriate permissions, configuring team access to tables and workflows, implementing permission middleware in integrations, setting up audit trails for access events, and managing organization-level settings. Essential for enterprise teams with multiple users and sensitive prospect data.
