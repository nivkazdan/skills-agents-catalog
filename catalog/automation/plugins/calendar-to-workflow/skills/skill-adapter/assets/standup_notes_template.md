# Standup Notes Template

Use this template to generate consistent and effective standup notes. Fill in the placeholders below with your individual updates.

## Date: `YYYY-MM-DD`

## Team: `Team Name`

## Individual: `Your Name`

---

### 1. Yesterday's Accomplishments

*   [ ] **Task 1:** Describe what you completed yesterday. Be specific and quantify your progress whenever possible.
    *   *Example:*  Finished implementing the user authentication module for the new web application.  Successfully integrated with the existing database.
*   [ ] **Task 2:**  Another accomplishment.
    *   *Example:*  Reviewed and merged 3 pull requests from the team.
*   [ ] **Task 3:**  Yet another accomplishment.
    *   *Example:*  Attended the daily scrum meeting and provided updates on my progress.

### 2. Today's Plan

*   [ ] **Task 1:** Describe what you plan to accomplish today. Be specific and realistic.
    *   *Example:*  Work on implementing the password reset functionality. Will focus on front-end integration.
*   [ ] **Task 2:**  Another task for today.
    *   *Example:*  Investigate a reported bug in the payment processing system.
*   [ ] **Task 3:**  Yet another task for today.
    *   *Example:*  Write unit tests for the user authentication module.

### 3. Blockers/Challenges

*   List any blockers or challenges you are facing. Be specific about what is preventing you from making progress. If you don't have any blockers, state "None."
    *   *Example:*  Still waiting for feedback on the API design from the backend team, blocking progress on the integration.
    *   *Example:*  Experiencing intermittent network connectivity issues, which are impacting my ability to run tests.
    *   *Example:*  None.

### 4. Assistance Needed

*   Specify if you need assistance from anyone on the team. Be specific about what kind of help you need and who you think can provide it. If you don't need assistance, state "None."
    *   *Example:*  Need help from John to troubleshoot the database connection issue.
    *   *Example:*  Would appreciate a code review on the authentication module from Mary.
    *   *Example:*  None.

### 5. Other Notes

*   Include any other relevant information, such as upcoming deadlines, important meetings, or general updates.
    *   *Example:*  Project demo scheduled for Friday afternoon.
    *   *Example:*  Taking PTO next Monday.
    *   *Example:*  Need to coordinate with Sarah on the marketing campaign launch next week.
---

**Instructions:**

1.  Fill out all the sections with accurate and relevant information.
2.  Use bullet points for clarity.
3.  Keep your updates concise and to the point.
4.  Be honest about your progress and challenges.
5.  Review your notes before sharing them with the team.
