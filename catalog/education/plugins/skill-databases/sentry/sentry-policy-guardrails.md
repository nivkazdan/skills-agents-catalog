## File Descriptions

### 1. SKILL.md
**Purpose:** Implement governance and policy guardrails for Sentry including shared configuration packages, compliance enforcement, and audit procedures.
**Workflow:** Implement at the organizational level. Creates standards and enforcement mechanisms for consistent Sentry usage across teams.
**Relates to:** Works with `sentry-enterprise-rbac` for access control. Connects to `sentry-data-handling` for compliance.

## Summary

This skill implements organizational policies and guardrails for Sentry usage. It covers shared configuration packages with enforced defaults, compliance enforcement with required data scrubbing, environment enforcement, standard alert templates, project naming standards, configuration audit scripts, and compliance dashboards. Use this skill when establishing organizational Sentry standards.
