## File Descriptions

### 1. SKILL.md
**Purpose:** Analyze, plan, and execute Clay API upgrades with breaking change detection. Covers version comparison, migration planning, and rollback procedures.
**Workflow:** Maintenance skill. Use when Clay releases API updates or when upgrading integration versions.
**Relates to:** Supports long-term maintenance of all Clay integrations; integrates with clay-ci-integration for testing.

## Summary

This skill guides users through Clay API upgrade processes. It covers analyzing Clay changelog for breaking changes, identifying affected code paths, creating migration plans with prioritized changes, implementing and testing updates, documenting rollback procedures, and coordinating with team members on upgrade timing. Essential for keeping Clay integrations current and secure.
